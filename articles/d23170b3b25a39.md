# 1. はじめに

家に眠っていたRaspberry Piを活用して、ミニKubernetesクラスタを構築してみました。  
正直「小さいRaspberry PiでK8sなんて動くのか？」と思っていたのですが、思った以上にサクッと動いて面白かったので、今回はその構築手順とハマりどころをまとめておきます。

「とりあえずクラスタを触ってみたい」「ローカルでK8sを学びたい」という人にもおすすめの構成です。

---

# 2. 用意したもの

今回は以下の構成で組みました。

- **Raspberry Pi 4 × 3台**（各4GBモデル）  
- **PoE HAT × 3つ**（電源供給をLANケーブルでまとめたかったので採用）  
- **PoE対応スイッチングハブ × 1台**  
- **LANケーブル × 3本**  
- **microSDカード（32GB） × 3枚**  

LANケーブル1本で電源＋通信がまかなえるので、机の上がスッキリしました。  
ネットワーク構成は単純にPoEハブに3台を有線接続しています。

# 3. Kubernetesクラスタの構築

OSは **Raspberry Pi OS Lite (64bit)** を使用。  
まず、3台それぞれにOSをインストールし、SSHでログインできるように設定。

| 役割 | ホスト名 | IPアドレス |
|------|-----------|-------------|
| master | k8s-first | 192.168.1.10 |
| node1 | k8s-second | 192.168.1.11 |
| node2 | k8s-third| 192.168.1.12 |

os情報
```
PRETTY_NAME="Debian GNU/Linux 13 (trixie)"
NAME="Debian GNU/Linux"
VERSION_ID="13"
VERSION="13 (trixie)"
VERSION_CODENAME=trixie
DEBIAN_VERSION_FULL=13.4
ID=debian
HOME_URL="https://www.debian.org/"
SUPPORT_URL="https://www.debian.org/support"
BUG_REPORT_URL="https://bugs.debian.org/"
```
## 3.1 swapの停止
zramが使われていたので、zramを無効化する。

```bash
$ sudo swapoff /dev/zram0
$ sudo apt remove systemd-zram-generator

$ swapon --show 
```
これで何も返ってこなかったらOK

## 3.2 コンテナランタイムの導入
今回は「Docker」を採用。

https://docs.docker.com/engine/install/debian/

## 3.2 kubernetesの導入
公式サイトをもとにkubeadm、kubelet、kubectlをインストール。
コマンドは公式サイトのものを使うように。

https://kubernetes.io/docs/setup/production-environment/tools/kubeadm/install-kubeadm/

### a. 必要なものをインストール
```bash
$ sudo apt update
$ sudo apt install -y apt-transport-https ca-certificates curl gpg
```

### b. 鍵の取得 
```bash
$ curl -fsSL https://pkgs.k8s.io/core:/stable:/v1.36/deb/Release.key | sudo gpg --dearmor -o /etc/apt/keyrings/kubernetes-apt-keyring.gpg
```
/etc/apt/keyrings がない場合は、以下のコマンドで作成
```bash
$ sudo mkdir -p -m 755 /etc/apt/keyrings
```

### c. aptリポジトリの登録
```bash
$ echo 'deb [signed-by=/etc/apt/keyrings/kubernetes-apt-keyring.gpg] https://pkgs.k8s.io/core:/stable:/v1.36/deb/ /' | sudo tee /etc/apt/sources.list.d/kubernetes.list
```

### d. インストール
インストールしてバージョンが勝手に上がらないように固定
```bash
$ sudo apt update
$ sudo apt install -y kubelet kubeadm kubectl
$ sudo apt-mark hold kubelet kubeadm kubectl
```

## 3.3 systemd cgroupドライバーを構成

https://kubernetes.io/ja/docs/setup/production-environment/container-runtimes/#containerd-systemd

を参考にして、
`/etc/containerd/config.toml` を修正。
```diff 
[plugins.'io.containerd.cri.v1.runtime'.containerd.runtimes.runc]
  ...
  [plugins.'io.containerd.cri.v1.runtime'.containerd.runtimes.runc.options]
-    SystemdCgroup = false
+    SystemdCgroup = true
```

containerd のサービスを再起動して適用
```bash
$ sudo systemctl restart containerd
```

## 3.4 Raspberry Pi の memory cgroup を有効化

`/boot/firmware/cmdline.txt` を編集
```bash
$ sudo nano /boot/firmware/cmdline.txt
```

末尾に下記を追記
```bash
cgroup_enable=cpuset cgroup_enable=memory cgroup_memory=1
```
保存後再起動
```bash
$ sudo reboot
```

## 3.1 kubeadmを使ったセットアップ

まずmasterノードで以下を実行して初期化。

```bash
sudo kubeadm init --pod-network-cidr=10.244.0.0/16
```

出力された`kubeadm join`コマンドをメモしておき、他の2台（node1, node2）で実行します。

```bash
sudo kubeadm join 192.168.1.10:6443 --token xxxxx --discovery-token-ca-cert-hash sha256:xxxxxx
```

## 3.2 ネットワークプラグインの導入

クラスタ内通信のために **Flannel** を導入

```bash
kubectl apply -f https://raw.githubusercontent.com/coreos/flannel/master/Documentation/kube-flannel.yml
```

`kubectl get nodes` で3台すべてが `Ready` になればクラスタ完成

```bash
kubectl get nodes
NAME         STATUS   ROLES           AGE   VERSION
k8s-first     Ready    control-plane   5m    v1.30.0
k8s-second    Ready    <none>          3m    v1.30.0
k8s-third     Ready    <none>          3m    v1.30.0
```

# 4. nginxのデプロイ

動作確認として、nginxをデプロイしてみます。

```bash
kubectl create deployment nginx --image=nginx
kubectl expose deployment nginx --port=80 --type=NodePort
```

NodePortで公開されたポートを確認：

```bash
kubectl get svc
```

ブラウザで  
`http://192.168.1.10:<NodePort>` にアクセスすると、  
おなじみの「Welcome to nginx!」が表示されます 🎉

# まとめ

- Raspberry Pi 3台でもKubernetesクラスタは普通に構築できる  
- PoEで電源まわりを整理できるのが便利  
- 軽めのアプリ（nginxなど）なら十分動く  

Raspberry Piクラスタは実運用向きではありませんが、**K8sを学ぶには最高の教材**です。  
今後はこの環境で、HelmやPrometheus、Ingress Controllerなども試していこうと思います。
